Accessibility Glossary

A comprehensive reference of accessibility terminology, definitions, and concepts.

About This Glossary

This glossary contains key terms and concepts related to digital accessibility, particularly web accessibility. Use it as a reference to better understand accessibility guidelines, practices, and technologies.

Navigate through the glossary using the alphabetical navigation below or use the search function to find specific terms.

A

  • Accessibility

    The design and development of digital content (websites, apps, documents) so that people with disabilities can perceive, understand, navigate, interact with, and contribute to it. Web accessibility means that people with disabilities can use the web equally.

    Related: WCAG ARIA Universal Design
  • Accessible Name

    The name of a user interface element as exposed to assistive technologies. The accessible name is calculated using various attributes and elements, such as labels, aria-label, aria-labelledby, and content, depending on the element.

    Related: aria-label aria-labelledby
  • Alternative Text (Alt Text)

    Text that serves as a semantic equivalent for non-text content, such as images. Alt text is read by screen readers to describe images to users who cannot see them. Well-written alt text should concisely convey the content and function of the image.

    Related: Non-text Content Screen Reader
  • ARIA (Accessible Rich Internet Applications)

    A set of attributes that define ways to make web content and web applications more accessible to people with disabilities. ARIA attributes can be added to HTML to provide additional information about the purpose, state, and functionality of elements to assistive technologies.

    Related: ARIA Landmark ARIA Live Regions WAI-ARIA
  • Assistive Technology (AT)

    Hardware or software that is used to increase, maintain, or improve the functional capabilities of individuals with disabilities. Examples include screen readers, screen magnifiers, alternative keyboards, switches, and voice recognition software.

    Related: Screen Reader Screen Magnifier

C

  • Caption (Closed Caption)

    Text synchronized with multimedia content (video, animation) that provides a textual alternative to the audio content. Captions not only include dialogue but also identify speakers and include non-speech information like sound effects and music.

    Related: Transcript Audio Description
  • Color Contrast

    The difference in perceived brightness between two colors, typically measured as a ratio. Adequate color contrast between text and its background is essential for readability, especially for users with low vision or color vision deficiencies.

    WCAG 2.1 requires a contrast ratio of at least 4.5:1 for normal text and 3:1 for large text at Level AA.

    Related: Color Vision Deficiency WCAG

K

  • Keyboard Accessibility

    The ability to navigate and interact with a website or application using only a keyboard, without requiring a mouse or other pointing device. Keyboard accessibility is essential for users with motor disabilities who cannot use a mouse and for those who use alternative input devices that emulate keyboard functionality.

    Related: Focus Keyboard Trap tabindex
  • Keyboard Trap

    A situation where keyboard focus gets stuck on a particular element, making it impossible for the user to navigate away using only the keyboard. Keyboard traps prevent users who rely on keyboard navigation from accessing all parts of a website or application.

    Related: Keyboard Accessibility Focus

S

  • Screen Reader

    Assistive technology that converts text and other visual elements on a screen to speech or braille output. Screen readers enable blind and low-vision users to access and interact with digital content by converting visual information into non-visual formats.

    Common screen readers include JAWS, NVDA, VoiceOver, and TalkBack.

    Related: Assistive Technology Alternative Text ARIA
  • Semantic HTML

    The use of HTML elements that convey meaning about the structure and content of a web page, rather than just its appearance. Semantic HTML provides important information to browsers, search engines, and assistive technologies about the purpose and relationships of different page elements.

    Examples of semantic elements include <header>, <nav>, <main>, <article>, <section>, and <footer>.

    Related: Accessibility ARIA Screen Reader

W

  • WCAG (Web Content Accessibility Guidelines)

    A set of guidelines developed by the Web Accessibility Initiative (WAI) of the World Wide Web Consortium (W3C) that explain how to make web content more accessible to people with disabilities. WCAG is organized around four principles: Perceivable, Operable, Understandable, and Robust (POUR).

    The current version is WCAG 2.1, with three levels of conformance: A (minimum), AA (recommended), and AAA (enhanced).

    Related: W3C WAI Accessibility
