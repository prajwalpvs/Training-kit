Content First Approach

Accessibility is most effective when considered from the beginning of the content creation process, not as an afterthought. When you plan your content with accessibility in mind, you'll create better experiences for all users.

Document Structure

A well-structured document provides a clear roadmap for all users, especially those using assistive technologies. Here's how to create proper document structure:

1

Use Proper Heading Hierarchy

Headings create a table of contents for your page, helping users understand its organization and navigate efficiently. Screen reader users often navigate by headings.

Heading Structure Example

Do
<h1>Main Page Title</h1>
<h2>First Section</h2>
<p>Content here...</p>
<h3>Subsection</h3>
<p>More content...</p>
<h2>Second Section</h2>

Uses a logical heading hierarchy that reflects the content structure.

Don't
<div class="title">Main Page Title</div>
<h3>First Section</h3>
<p>Content here...</p>
<h2>Subsection</h2>
<h4>Second Section</h4>

Uses headings based on appearance rather than hierarchy, skipping levels and using non-semantic elements.

  • Start with a single h1 element that describes the page's main topic
  • Don't skip heading levels (e.g., don't jump from h2 to h4)
  • Use headings to outline content, not for styling text
  • Keep headings concise and descriptive
2

Use Semantic HTML

Semantic HTML communicates meaning to browsers and assistive technologies, making content more accessible and easier to style and maintain.

Semantic HTML Example

Do
<article>
<header>
<h2>Article Title</h2>
<p>Posted on <time datetime="2023-06-15">June 15, 2023</time></p>
</header>
<p>Introduction paragraph...</p>
<section>
<h3>Section Heading</h3>
<p>Section content...</p>
</section>
<footer>
<p>Author: Jane Smith</p>
</footer>
</article>
Don't
<div class="article">
<div class="header">
<div class="title">Article Title</div>
<div class="date">Posted on June 15, 2023</div>
</div>
<div class="content">
<div>Introduction paragraph...</div>
<div class="section">
<div class="section-title">Section Heading</div>
<div>Section content...</div>
</div>
</div>
<div class="footer">
<div>Author: Jane Smith</div>
</div>
</div>
  • Use semantic elements like article, section, nav, header, footer, etc.
  • Use appropriate list elements (ul, ol, dl) for list content
  • Use blockquote for extended quotations
  • Use figure and figcaption for images with captions
  • Use table for tabular data (not for layout)
3

Create Logical Reading Order

Content should be presented in a logical, intuitive sequence that makes sense when read aloud. Screen readers navigate through content in the order it appears in the HTML, not as it's visually arranged with CSS.

  • Structure HTML to follow a logical reading order from top to bottom
  • Don't rely solely on visual layout to convey relationships between content
  • Test your content with a screen reader to ensure it reads in a sensible order
  • Be careful with CSS positioning that changes visual order without changing the HTML order

Text Content

The way you write and format text has a significant impact on accessibility. Here are best practices for creating accessible text content:

4

Write Clear, Simple Content

Clear, straightforward language benefits all users but is especially important for those with cognitive disabilities or who use translation tools.

  • Use plain language and avoid jargon when possible
  • Break up long paragraphs into shorter chunks
  • Use bulleted or numbered lists for complex information
  • Provide definitions for technical terms
  • Use active voice most of the time
5

Ensure Sufficient Color Contrast

Color contrast affects the readability of text for all users, especially those with low vision or color blindness.

Color Contrast Demonstration

This text demonstrates the visual contrast between text and background colors.
  • Normal text needs a contrast ratio of at least 4.5:1
  • Large text (18pt+ or 14pt+ bold) needs a contrast ratio of at least 3:1
  • Use contrast checking tools to verify your color combinations
  • Don't use color as the only way to convey information
6

Make Text Resizable

Users should be able to enlarge text without breaking the layout or losing content.

  • Use relative units like em, rem, or percentages for text sizes
  • Avoid using fixed-size text that can't be resized by the browser
  • Test your content by zooming to 200% to ensure it remains readable
  • Ensure line height is sufficient for readability when text is enlarged

Images & Graphics

Visual content needs text alternatives to be accessible to people who can't see it.

7

Write Effective Alt Text

Alternative text provides a textual alternative to non-text content like images, diagrams, and charts.

Alt Text Examples

Do

Image: Company logo

<img src="logo.png" alt="Acme Corporation logo">

Image: Chart showing data

<img src="sales-chart.png" alt="Bar chart showing sales increasing by 25% in Q2 2023">

Image: Decorative divider

<img src="divider.png" alt="">
Don't

Image: Company logo

<img src="logo.png"> (Missing alt text)

Image: Chart showing data

<img src="sales-chart.png" alt="Chart"> (Too vague)

Image: Decorative divider

<img src="divider.png" alt="Decorative line"> (Should be empty for decorative images)
  • Be concise but descriptive about the purpose or content of the image
  • For complex images like charts or graphs, include the key information they convey
  • For decorative images that don't add meaning, use empty alt text (alt="")
  • Don't start alt text with "Image of" or "Picture of" (screen readers already announce it's an image)
  • For linked images, describe the destination or purpose of the link in the alt text
8

Handle Complex Images

Complex images like infographics, diagrams, or complex charts need more detailed descriptions.

  • Provide a brief alt text summary and a more detailed description elsewhere
  • For complex diagrams, consider using the figure and figcaption elements
  • For infographics, provide the same information in an accessible text format
  • Complex charts might need a separate table of data and a text description

Links & Navigation

Links and navigation elements should be clear and usable by all visitors.

9

Write Descriptive Link Text

Link text should make sense out of context and clearly indicate where the link will take the user.

Link Text Examples

Do
<a href="policy.html">Read our privacy policy</a>

<a href="report.pdf">Download the 2023 Annual Report (PDF, 2.5MB)</a>
Don't
<a href="policy.html">Click here</a> to read our privacy policy

<a href="report.pdf">Here</a>
  • Avoid vague link text like "click here," "read more," or "learn more"
  • Make link text that can stand on its own and describes the destination
  • Indicate if links open in a new window or download files
  • Include file type and size information for downloadable files
  • Don't use the same link text for different destinations
10

Provide Skip Links

Skip links help keyboard users bypass repetitive navigation menus and jump directly to the main content.

Skip Link Example

<body>
<a href="#main-content" class="skip-link">Skip to main content</a>
<header>...</header>
<nav>...</nav>
<main id="main-content">
<h1>Page Title</h1>
...
</main>
</body>
  • Place skip links at the beginning of the page
  • Make skip links visible when focused for keyboard users
  • Ensure the target of the skip link has a tabindex="-1" if it's not naturally focusable
  • Consider adding multiple skip links for pages with complex structures

Tables & Data

Tables should be used for data, not layout, and need proper structure to be accessible.

11

Structure Tables Properly

Well-structured tables help screen reader users understand the relationships between data cells and headers.

Accessible Table Example

<table>
<caption>Quarterly Sales by Region, 2023</caption>
<thead>
<tr>
<th scope="col">Region</th>
<th scope="col">Q1</th>
<th scope="col">Q2</th>
<th scope="col">Q3</th>
<th scope="col">Q4</th>
</tr>
</thead>
<tbody>
<tr>
<th scope="row">North</th>
<td>$10,000</td>
<td>$12,500</td>
<td>$15,000</td>
<td>$18,000</td>
</tr>
<!-- Additional rows -->
</tbody>
</table>
  • Include a caption that describes the table's content
  • Use thead, tbody, and tfoot to group rows
  • Use th for header cells with appropriate scope attributes
  • For complex tables, use id and headers attributes to associate data cells with headers
  • Avoid merged cells when possible as they complicate screen reader navigation

Forms & Interactions

Forms should be accessible to all users, including those using assistive technologies.

12

Label Form Controls

All form controls need properly associated labels so users understand what information is required.

Form Labeling Examples

Do
<label for="name">Full Name</label>
<input type="text" id="name" name="name">

<fieldset>
<legend>Subscription Type</legend>
<input type="radio" id="basic" name="subscription">
<label for="basic">Basic</label>
<input type="radio" id="premium" name="subscription">
<label for="premium">Premium</label>
</fieldset>
Don't
Full Name
<input type="text" name="name">

Subscription Type
<input type="radio" name="subscription"> Basic
<input type="radio" name="subscription"> Premium
  • Use the label element with a for attribute that matches the input's id
  • Group related form controls with fieldset and legend
  • Provide instructions and expectations (like required fields or format constraints)
  • Ensure labels are descriptive and concise
13

Provide Clear Error Messages

Error messages should help users identify and fix problems with their form submissions.

Error Message Example

Do
<label for="email">Email Address</label>
<input type="email" id="email" name="email" aria-describedby="email-error" aria-invalid="true">
<p id="email-error" class="error">Please enter a valid email address in the format name@example.com</p>
Don't
<input type="email" name="email" class="error">
<p class="error">Invalid!</p>
  • Be specific about what went wrong and how to fix it
  • Use aria-describedby to associate error messages with form fields
  • Use aria-invalid="true" on fields with errors
  • Provide both visual and programmatic indication of errors
  • Ensure error messages are announced by screen readers

Tools for Content Creators

These tools can help you evaluate and improve the accessibility of your content:

Color Contrast Tools

These tools help check if your text and background colors meet WCAG contrast requirements:

  • WebAIM Contrast Checker - A simple tool for checking color contrast ratios
  • Colour Contrast Analyser - Desktop application for checking color contrasts
  • Accessible Color Palette Builder - Create color palettes that meet contrast requirements
Try WebAIM Contrast Checker

Text Analysis Tools

These tools help you evaluate and improve your text content:

  • Hemingway Editor - Highlights complex sentences, passive voice, and difficult reading levels
  • Grammarly - Checks grammar and readability
  • Readability Test Tool - Analyzes text for readability scores
Try Hemingway Editor

Image Alt Text Tools

These tools help you create better alt text for images:

  • Alt Text Decision Tree - Guides you through decisions about appropriate alt text
  • WebAIM Alternative Text Guide - Comprehensive guide to writing effective alt text
  • Alt Text Tester - Test how your alt text sounds with a screen reader
Try Alt Text Decision Tree

General Accessibility Checkers

These tools help identify a range of accessibility issues:

  • WAVE - Web Accessibility Evaluation Tool from WebAIM
  • Axe - Accessibility testing engine and browser extensions
  • Lighthouse - Built into Chrome DevTools
Try WAVE
