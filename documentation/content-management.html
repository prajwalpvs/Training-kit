Accessible Content Management Process

A comprehensive guide to creating, reviewing, and maintaining accessible content throughout its lifecycle.

Overview

This document outlines a standardized process for managing accessible content from planning to publication and maintenance. Following this process ensures that accessibility is integrated throughout the content lifecycle rather than treated as an afterthought.

The process is designed to be flexible enough to adapt to different types of content while providing clear guidelines and checkpoints to maintain accessibility standards. By integrating accessibility from the beginning, you can save time, reduce costs, and ensure all users can access your content.

Key Principles
  • Accessibility by design: Include accessibility considerations from the planning stage
  • Test early and often: Identify accessibility issues throughout the process
  • Continuous learning: Regularly update knowledge about accessibility best practices
  • Clear responsibilities: Everyone involved understands their role in creating accessible content

1. Planning

1

Define Content Requirements

Start by clearly defining the purpose, audience, and objectives of the content. Consider accessibility requirements from the beginning of the planning process.

Planning Checklist
  • Identify target audience, including users with disabilities
  • Define the minimum accessibility level (e.g., WCAG 2.1 Level AA)
  • Document specific accessibility requirements for the content type
  • Identify content that requires alternatives (videos, images, etc.)
  • Allocate resources for creating accessible alternatives
2

Create Content Structure

Develop a clear content structure that supports accessibility, including headings, sections, and navigation that will make the content easy to navigate and understand.

Structure Checklist
  • Plan logical heading structure (H1-H6)
  • Organize content in a clear, sequential order
  • Identify areas that need lists, tables, or other structured elements
  • Plan navigation aids for longer content
3

Select Accessible Templates and Tools

Choose content templates, authoring tools, and resources that support accessibility features and make it easier to create accessible content.

Template and Tools Checklist
  • Select pre-approved accessible templates where available
  • Verify that authoring tools support accessibility features
  • Ensure templates include proper semantic structure
  • Configure accessibility checking tools

2. Creation

1

Develop Content Following Accessibility Guidelines

Create content with accessibility in mind, implementing best practices for your content type.

Content Creation Checklist
  • Write clear, simple content with plain language
  • Use proper heading structure (H1-H6)
  • Provide descriptive link text (no "click here" or "read more")
  • Add meaningful alt text for all non-decorative images
  • Use proper list markup for lists
  • Create proper table structure with headers
  • Ensure sufficient color contrast
  • Do not use color alone to convey information
2

Create Accessible Multimedia

Develop accessible alternatives for multimedia content such as videos, audio, and interactive elements.

Multimedia Checklist
  • Create captions for videos
  • Provide transcripts for audio content
  • Add audio descriptions for videos when needed
  • Ensure interactive elements are keyboard accessible
  • Provide alternatives for complex interactions
3

Initial Self-Testing

Perform basic accessibility checks during content creation to catch issues early.

Self-Testing Checklist
  • Use built-in accessibility checkers in authoring tools
  • Review content with the accessibility checklist
  • Test keyboard navigation for interactive elements
  • Verify reading order makes logical sense
  • Check readability level of content

3. Review

1

Automated Accessibility Testing

Use automated tools to check for common accessibility issues that can be detected programmatically.

Automated Testing Checklist
  • Run content through appropriate accessibility checking tools
  • Document all issues identified by automated tests
  • Prioritize issues by severity and impact
  • Verify false positives and document for review
2

Manual Accessibility Review

Perform a detailed manual review to catch issues that automated tools might miss and verify the user experience.

Manual Review Checklist
  • Review all alt text for accuracy and usefulness
  • Verify logical reading and navigation order
  • Test keyboard accessibility for all interactive elements
  • Check color contrast with tools and visual inspection
  • Verify that form elements have proper labels
  • Ensure content is understandable and logically organized
3

Assistive Technology Testing

Test content with assistive technologies to ensure it works properly for users with disabilities.

Assistive Technology Testing Checklist
  • Test with screen readers (e.g., NVDA, VoiceOver, JAWS)
  • Test with keyboard navigation only
  • Test with screen magnification
  • Verify functionality with speech recognition
  • Document issues experienced with assistive technologies

4. Remediation

1

Prioritize Issues

Organize identified accessibility issues by severity, impact, and complexity to create an effective remediation plan.

Issue Prioritization Checklist
  • Categorize issues by WCAG conformance level (A, AA, AAA)
  • Identify issues that completely block access for certain users
  • Assess effort required to fix each issue
  • Develop a timeline for addressing all issues
2

Fix Accessibility Issues

Address identified issues following best practices and accessibility guidelines.

Issue Resolution Checklist
  • Fix critical Level A issues first
  • Implement fixes according to WCAG guidelines
  • Document all changes made
  • Consult accessibility experts for complex issues
3

Verification Testing

Re-test after fixes to ensure issues have been properly addressed and no new issues have been introduced.

Verification Testing Checklist
  • Re-test each fixed issue to confirm resolution
  • Perform regression testing to ensure no new issues
  • Verify fixes work across different browsers and devices
  • Retest with assistive technologies

5. Publication

1

Final Accessibility Review

Conduct a final check of the content in its publication environment to ensure all accessibility features work as expected.

Final Review Checklist
  • Verify content accessibility in the actual publishing environment
  • Check that accessibility features weren't affected by the publishing process
  • Test with multiple browsers and devices
  • Ensure content integrates properly with the surrounding website or platform
2

Prepare Accessibility Documentation

Document the accessibility features of the content and any known limitations for transparency.

Documentation Checklist
  • Create an accessibility statement if needed
  • Document any known accessibility limitations
  • Document alternative formats or access methods available
  • Provide contact information for accessibility support
3

Publish Content

Release the accessible content to users, ensuring all accessibility features are properly included.

Publication Checklist
  • Ensure all accessibility-related files are included (captions, transcripts, etc.)
  • Verify metadata includes accessibility information
  • Make sure accessibility documentation is published alongside content
  • Test the published content one final time

6. Maintenance

1

Monitor Accessibility Feedback

Actively collect and respond to feedback about accessibility issues from users.

Feedback Monitoring Checklist
  • Provide clear channels for accessibility feedback
  • Regularly review accessibility complaints and issues
  • Prioritize and address reported accessibility problems
  • Document common issues for future content improvements
2

Conduct Regular Accessibility Audits

Perform periodic checks to ensure content remains accessible over time, especially after updates or changes to the surrounding platform.

Audit Checklist
  • Schedule regular accessibility reviews
  • Recheck content after platform updates
  • Review content against updated accessibility standards
  • Test with newer versions of assistive technologies
3

Update Content as Needed

Make ongoing improvements to content accessibility based on feedback, audits, and evolving accessibility standards.

Content Update Checklist
  • Apply accessibility improvements when updating content
  • Ensure any new content elements follow accessibility guidelines
  • Update accessibility documentation as needed
  • Communicate significant accessibility improvements to users

Roles and Responsibilities

Clear definition of roles and responsibilities is essential for an effective accessibility process. Here's a breakdown of typical responsibilities:

Role Responsibilities
Content Creator
  • Follow accessibility guidelines when creating content
  • Add proper alt text, headings, and structure
  • Perform initial accessibility self-checks
  • Address basic accessibility issues
Content Editor/Reviewer
  • Review content for accessibility compliance
  • Use accessibility checkers during review
  • Provide feedback on accessibility issues
  • Verify compliance with accessibility standards
Accessibility Specialist
  • Provide expert guidance on complex accessibility issues
  • Conduct advanced accessibility testing
  • Test with assistive technologies
  • Stay updated on accessibility standards and best practices
Content Manager
  • Ensure accessibility is integrated into the content workflow
  • Allocate resources for accessibility needs
  • Monitor overall accessibility compliance
  • Address accessibility feedback from users
Developer/Technical Support
  • Implement technical accessibility solutions
  • Ensure content platforms support accessibility
  • Create accessible templates and components
  • Address technical accessibility issues
