Provide text alternatives for non-text content

Mark as completed
WCAG 1.1.1 Level A Perceivable

All non-text content (images, icons, charts, etc.) must have a text alternative that serves the equivalent purpose.

Why this matters

Text alternatives are essential for people who cannot see images, such as screen reader users. They provide the same information conveyed by the image in an accessible format.

How to implement

  • Add descriptive alt text to images that convey meaning
  • Use empty alt text (alt="") for decorative images
  • Provide text alternatives for complex images like charts or diagrams
  • Ensure form image buttons have descriptive alt text

Examples

Good Example
Bad Example

Good example with descriptive alt text:

<img src="chart.png" alt="Bar chart showing sales growth of 25% in Q1, 15% in Q2, and 30% in Q3">

Bad example with missing or inadequate alt text:

<img src="chart.png" alt="Chart"> <img src="logo.png"> <!-- Missing alt attribute -->

Testing

  1. Turn off images in your browser or use a screen reader to check if the information conveyed by images is still available
  2. Inspect the HTML to ensure all images have appropriate alt attributes
  3. Verify that decorative images have empty alt text (alt="")

Provide captions for videos

Mark as completed
WCAG 1.2.2 Level A Perceivable

Captions must be provided for all prerecorded audio content in synchronized media.

Why this matters

Captions make video content accessible to people who are deaf or hard of hearing. They also benefit people in noisy environments, those who are not fluent in the language spoken, and people who process written information better than audio.

How to implement

  • Provide synchronized captions for all video content with audio
  • Ensure captions include all spoken dialogue and important sound effects
  • Use a video player that supports captions
  • Consider using platforms like YouTube that offer automatic captioning as a starting point (but always review and edit for accuracy)

Ensure all functionality is operable via keyboard

Mark as completed
WCAG 2.1.1 Level A Operable

All functionality must be operable through a keyboard interface without requiring specific timings for individual keystrokes.

Why this matters

Many users rely on a keyboard to navigate websites, including people with motor disabilities, blind users, and power users. Ensuring keyboard operability also improves compatibility with other assistive technologies.

How to implement

  • Ensure all interactive elements can be accessed and activated using Tab, Enter, and other keyboard controls
  • Avoid keyboard traps where focus can get stuck
  • Provide visible focus indicators
  • Don't rely on mouse-specific events like hover or drag without keyboard alternatives

Make text content readable and understandable

Mark as completed
WCAG 3.1.2 Level AA Understandable

The language of parts of content that differ from the default language of the page must be programmatically determined.

Why this matters

Identifying the language of content allows screen readers and other assistive technologies to process text correctly, ensuring proper pronunciation and understanding.

Examples

Good Example
Bad Example

Good example with language attributes:

<html lang="en"> <head>...</head> <body> <p>The French word for book is <span lang="fr">livre</span>.</p> </body> </html>

Bad example missing language attributes:

<html> <head>...</head> <body> <p>The French word for book is livre.</p> </body> </html>

Use valid, semantic HTML

Mark as completed
WCAG 4.1.1 Level A Robust

In content implemented using markup languages, elements have complete start and end tags, are nested according to their specifications, and do not contain duplicate attributes.

Why this matters

Valid, well-formed HTML helps ensure that user agents, including assistive technologies, can accurately interpret and parse content. Using semantic HTML also provides additional contextual information about the purpose and structure of content.

How to implement

  • Use HTML according to specification (proper nesting, complete tags, no duplicate attributes)
  • Validate your HTML using the W3C Validator
  • Use semantic HTML elements like <header>, <nav>, <main>, <section>, <article>, <footer>
  • Use <button> for clickable controls and <a> for navigation links
